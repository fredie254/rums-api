RUMS REST API

Rental Unit Management System — standalone JSON REST API. All responses are JSON. All protected endpoints require a Bearer token.

https://api.rums.ultimatesolutions.co.ke/api/v1

Request Format

Send Content-Type: application/json with a JSON body for POST/PUT/PATCH requests.

Response Envelope

{
  "success": true | false,
  "data":    { ... } | [ ... ],   // on success
  "meta":    { "total": 0, "per_page": 20, "current_page": 1, "total_pages": 1 },
  "message": "Human-readable message",
  "errors":  [ ... ],             // on validation failure
  "_ms":     12.4                 // response time in milliseconds
}

Roles & Scopes

RoleDefault Scopes
adminAll access
managerread/write: properties, units, tenants, leases, payments, invoices, maintenance, reports
accountantread: all · write: payments, invoices
landlordread: properties, units, leases, payments, invoices, reports
maintenanceread/write: maintenance · read: units
auditorread: all (no write)
tenantread: leases, payments, invoices · write: maintenance

Authentication

All protected endpoints require a Bearer token in the Authorization header.

Authorization: Bearer <your-token>
GET requests also accept ?api_token=<token> as a convenience parameter.
POST /auth/login Public Exchange credentials for a token
Request Body
FieldTypeRequiredDescription
emailstringrequiredUser email address
passwordstringrequiredUser password
Example Request
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "admin@rums.co.ke",
  "password": "Admin@2024!"
}
Example Response — 200
{
  "success": true,
  "data": {
    "token": "a3f9c2...",
    "token_type": "Bearer",
    "expires_in": 31536000,
    "user": { "id": 1, "name": "System Admin", "email": "admin@rums.co.ke", "role": "admin" }
  },
  "message": "Login successful."
}
POST /auth/logout Bearer Revoke current token

No body required. Revokes the token used in the Authorization header.

Example Response — 200
{ "success": true, "message": "Logged out." }
GET /auth/me Bearer Current user profile
Example Response — 200
{
  "success": true,
  "data": {
    "user":  { "id": 1, "name": "Admin", "email": "admin@rums.co.ke", "role": "admin", "status": "active", "last_login": "2024-06-01T08:00:00+03:00" },
    "token": { "id": 5, "name": "Login Token", "scopes": "admin", "last_used": "2024-06-01T09:00:00", "expires_at": "2025-06-01T08:00:00" }
  }
}
POST /auth/token Bearer Issue a named API token
FieldTypeDescription
namestringoptionalToken label (default: "API Token")
scopesstringrequiredComma-separated scopes e.g. read:properties,write:payments
expires_daysintoptionalTTL in days (0 = never expires)
DELETE /auth/token/{id} Bearer Revoke a token by ID (own tokens; admin can revoke any)

Properties

Manage rental properties. Requires read:properties / write:properties scope.

GET /properties Bearer List all properties
Query Parameters
ParamTypeDescription
searchstringFilter by name or address
statusstringactive | inactive
landlord_idintFilter by landlord
pageintPage number (default: 1)
per_pageintResults per page (default: 20, max: 200)
POST /properties Bearer Create a property
Body Fields
FieldTypeDescription
namestringrequiredProperty name
property_typestringrequirede.g. apartment, house, commercial
address_countystringrequiredCounty
address_line1stringoptionalStreet address
address_citystringoptionalCity / town
total_unitsintoptionalExpected unit count
landlord_idintoptionalLinked landlord ID
manager_idintoptionalLinked manager user ID
descriptionstringoptionalFree text
amenitiesstringoptionalFree text
GET /properties/{id} Bearer Single property with units + stats
PATCH /properties/{id} Bearer Update property fields

Send only the fields you want to change. Same fields as POST (except required fields become optional).

DELETE /properties/{id} Bearer Soft-delete a property
Fails if any units are still occupied.
PUT /properties/{id} Bearer Full update of a property

Same fields as PATCH. Use PUT to signal a full replacement of editable fields.

GET /properties/{id}/units Bearer All units for a property with current tenant
GET /properties/{id}/stats Bearer Occupancy and income stats for a property
Example Response
{
  "total_units": 12, "occupied": 10, "available": 2, "maintenance": 0,
  "potential_monthly_revenue": 120000,
  "year_income": 980000
}

Error Codes

HTTPMeaning
200OK
201Created
204No Content (OPTIONS preflight)
400Bad request — missing or invalid parameters
401Unauthorized — missing or invalid Bearer token
403Forbidden — insufficient role or scope
404Not found — resource or endpoint doesn't exist
405Method not allowed
409Conflict — e.g. duplicate email
422Unprocessable — validation failure with errors array
429Too many requests — rate limit exceeded
503Service unavailable — database unreachable
